# Employee Onboarding Guide

## Purpose

This document defines the official onboarding process for new employees.
It serves as the internal reference for HR, IT, and managers.

---

# Where is the Onboarding Documentation?

The onboarding documentation is stored in **Confluence**.

- **Confluence Space:** People Operations
- **Page Name:** Employee Onboarding (Day 0–30)
- **Example URL:** https://confluence.company.com/display/PEOPLE/Employee+Onboarding

If you do not have Confluence access:
- Submit request via **ServiceNow**
- Search for: "Access Request: Confluence"

This is the primary source of onboarding documentation.

---

# What is Onboarding?

Onboarding is the structured process of enabling a new employee to become productive, secure, and aligned with company standards.

It includes:

- Role context and architecture overview
- Required system access and permissions
- Training plan
- Compliance and security awareness
- Manager alignment
- Team integration

---

# Onboarding Process Overview

## Phase 1: Pre-Day 0

- HR profile creation
- Account provisioning (IAM, Email, Teams)
- Hardware setup
- Documentation access preparation
- Training scheduling

---

## Phase 2: Day 0 (First Login)

1. Sign in to IAM / SSO
2. Access corporate email (Outlook)
3. Join Microsoft Teams + required channels
4. Access Jira project boards
5. Access GitHub organization + team repositories
6. Access Confluence space + runbooks
7. Configure DevOps Deploy App (CI/CD)
8. Complete mandatory Security Awareness training

---

## Phase 3: Day 1 Checklist

- Welcome session with manager
- IT login verification
- Security briefing
- Team introduction
- Role expectation discussion
